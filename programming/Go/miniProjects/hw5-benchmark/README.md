# Проект №5

В проекте требуется реализовать два крупных блока функциональности:

1. Бенчмарк-тесты, сравнивающие производительность различных способов вызова методов и решения типовых задач (в зависимости от языка).
2. Утилиту inspector, которая анализирует структуру типа, строит иерархию наследования, выводит данные в текстовом и JSON-форматах, а также генерирует экземпляры классов.

## 1. Бенчмарки производительности

Бенчмарк-тесты обеспечивают объективный, измеримый ориентир для сравнения и оценки производительности. Они устанавливают четкие ожидания и позволяют проводить сравнительный анализ результатов.

В этом задании вам потребуется написать тесты и провести сравнительное испытание для разных способов вызова метода.

### Общие требования

* Написать набор бенчмарк-тестов для каждого варианта.
* Использовать профильный фреймворк измерения производительности: `testing`.
* Подготовительная работа должна выполняться вне тела итераций бенчмарка.
* Длительность прогонов должна позволять получить стабильные результаты.
* Опубликовать итоговую таблицу с результатами.

### Сценарий тестирования

Реализовать бенчмарки для четырёх вариантов вызова метода `Student.name()`:

* Прямой вызов
* Интерфейсный вызов
* [Method value / method expression](https://tutorial.xgo.dev/method-values-and-expressions)
* С помощью пакета reflect

### Инструкции по реализации

* Замеры производительности следует выполнять с использованием одинаковых вычислительных ресурсов, при низкой нагрузке на тестовую платформу.
* Весь код получения reflection-объектов, method handles, regexp, сериализаторов и т.д. вынести из тела итераций.
* Зафиксировать итоговую таблицу результатов в отдельном файле и приложить к основному MR.

#### Пример сформированной таблицы:

```text
BenchmarkStudentName_Direct-14    	1000000000	         0.2790 ns/op	       0 B/op	       0 allocs/op
PASS
```

## 2. Утилита Inspector

Утилита должна анализировать любую структуру, собирать информацию о ее полях, строить дерево наследования, а также создавать заполненные экземпляры классов.

### Функциональные требования

Программа должна представлять собой утилиту командной строки, которая принимает на вход следующие параметры:

* `-package`, `-p` go пакет, в котором находится структура
* `-struct`, `-s` - название структуры.
  Программа должна завершать свою работу с ошибкой, если структура по переданному имени не найдена
* `-format`, `-f` - необязательный параметр, указывающий на формат вывода результатов.
  * необходимо реализовать поддержку следующих форматов:
    * TEXT
    * JSON
  * утилита должна завершать свою работу с ошибкой, если на вход передан неподдерживаемый формат

При реализации не забывайте, что программа представляет собой консольную утилиту.
Поэтому, любое исключение в приложении должно транслироваться в соответствующий код возврата:

* 0 - программа успешно завершила свою работу
* 1 - непредвиденная ошибка
* 2 - некорректное использование программы (неверные параметры)

Пример запуска программы:
`inspector -file ./test_package --struct MyStruct --format TEXT`

### Анализ структуры

#### Сигнатура метода

`func Inspect[T any](, format string)`

#### Функционал анализа:

* Имя структуры (полное квалифицированное имя)
* Встроенные структуры (если есть)
* Поля:
  - имя
  - тип
  - field tags
* Методы:
  - имя
  - параметры (тип и порядок)
  - возвращаемый тип
* Иерархия встраивания в виде дерева

#### Форматы вывода:

##### **TEXT**

```
Class: Person
Embeds: Human
Fields:
  - name (String)
  - age (int)
Methods:
  - Name() : String
  - SetName(String)
  - Age() : int
  - setAge(int)
Hierarchy:
  Manager
    └── Employee
          └── Person
                 └── Human
```

##### **JSON**

```json
{
  "class": "Person",
  "superclass": "Human",
  "interfaces": ["Serializable"],
  "fields": [
    { "name": "name", "type": "String" },
    { "name": "age", "type": "int" }
  ],
  "methods": [
    { "name": "Name", "params": [], "returnType": "String" },
    { "name": "setName", "params": ["String"] },
    { "name": "Age", "params": [], "returnType": "int" },
    { "name": "setAge", "params": ["int"] }
  ],
  "annotations": ["Entity"],
  "hierarchy": {
      "Manager": {
        "Employee": {
          "Person": {
            "Human": {}
        }
      }
    }
  }
}
```

### Создание экземпляров классов

Метод должен создавать инстанс типа и автоматически заполнять его поля случайными значениями
`func Create[T any]() T`.


### Нефункциональные требования

Форматы вывода должны строго соответствовать описанному шаблону.
Генерация значений должна поддерживать:
- строки
- числа
- булевые
- даты
- вложенные объекты
- коллекции/списки/массивы

### Инструкции по реализации

#### Анализ структуры

* Использовать AST parser для инспектора:
* Форматирование вывода
  * TEXT: использовать `strings.Builder` для построчного формирования
  * JSON: создать структуру и сериализовать (использовать пакет `encoding/json`)

#### Создание экземпляров

Алгоритм создания нового объекта:

- создавать с помощью рефлексии экземпляр типа
- заполнять примитивы и строки случайными значениями
- рекурсивно создавать вложенные объекты
- ограничивать глубину вложенности
- работать только с публичными полями
