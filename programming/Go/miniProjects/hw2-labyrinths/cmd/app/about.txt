Инструкция по использованию генератора и, по совместительству с тем, решателя лабиринтов:

Лабиринт можно сгенерировать или решить. Используй для этого соответственно generate и solve параметры
Для генерации лабиринта используй одну из следующих стратегий:
dfs, prim, kruskal
Для решения одну из этих:
digger, astar, bfs, dijkstra
Используй для указания алгоритма флаг --algorythm
Результат выводтится в файл, который указан в --output (по умолчанию выводится в консоль)
Если указать -unicode, то выведется красивая картинка с символами unicode
Как параметры для генерации лабиринта используются введённые длина и ширина поля
В качестве параметров для решения лабиринта используются файл, где находится лабиринт, стартовая точка и конечная точка. 
Будь внимателен: эти точки не могут быть вне лабиринта или на стене лабиринта!
Координаты точек перечисляй через запятую и исполььзуй для этого соответствующий флаги: --start, --end
Для файла с лабиринтом используй флаг --file

Внимание! 
Если ты не хочешь генерировать лабиринт с особенными полями (песок и монеты), то при генерации используй флаг -unicode. 
Поле будет сохранено в сжатом формате и без особых полей.
Если хочешь сгенерировать лабиринт с особенными полями (песок и монеты), то генерируй лабиринт без флага -unicode

Если хочешь найти оптимальный путь, учитывая песок (затруднение) и монетки (улучшение), то запусти алгоритм "optimal"
